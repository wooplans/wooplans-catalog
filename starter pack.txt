## 1. VUE D'ENSEMBLE DU PROJET

### 1.1 Concept métier
WooPlans V2.2 adopte un modèle **"Starter Pack"** à 14 900 FCFA (prix barré 30 000 FCFA, -50%) :
- Achat unique d'un plan architectural spécifique
- Livraison immédiate par PDF composite personnalisé
- Upsell manuel vers plans complets (150k-450k FCFA) via WhatsApp

### 1.2 Contenu du PDF livré
Le document PDF comprend :
1. **Page couverture** : Images 3D, nom du client, date
2. **Plan d'implantation** (si fourni)
3. **Plan de distribution** (PNG haute résolution)
4. **Devis estimatif** : 2 gammes (Moyen/Haut standing) avec fourchettes
5. **Rendus 3D** extérieurs
6. **Page Upsell** : upsell vers plan complet

### 1.3 Stack technique
- **Frontend** : Next.js 15 (App Router), React 19, TypeScript, Tailwind CSS
- **Backend** : Next.js API Routes, Prisma ORM
- **Base de données** : PostgreSQL (Supabase/Neon)
- **Stockage fichiers** : Bunny CDN
- **Paiement** : Chariow (Mobile Money MTN/Orange)
- **Notifications** : Green API (WhatsApp Business)
- **Email** : Resend
- **PDF** : pdf-lib (Node.js)

## 3. SCHÉMA DE BASE DE DONNÉES

### 3.1 Modèle Prisma complet

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

enum PlanType {
  VILLA
  DUPLEX
}

enum PurchaseStatus {
  PENDING
  COMPLETED
  FAILED
  REFUNDED
}

enum UpsellStatus {
  NOT_CONTACTED
  CONTACTED
  QUOTE_SENT
  ADVANCE_PAID
  COMPLETED
  LOST
}

model Plan {
  id              String    @id @default(cuid())
  name            String
  type            PlanType
  surface         Int
  status          String    @default("ACTIVE")
  coverImageUrl   String    // Image 3D couverture
  implantationUrl String?   // Optionnel
  distributionUrl String    // Plan PNG
  render3dUrls    String[]  // Array URLs
  basePdfUrl      String?   // Template PDF
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt
  purchases       Purchase[]
}

model Purchase {
  id                String         @id @default(cuid())
  planId            String
  plan              Plan           @relation(fields: [planId], references: [id])
  
  // Données client
  fullName          String
  email             String
  phoneNumber       String
  
  // Transaction
  chariowReference  String?        @unique
  amount            Int            @default(14900)
  currency          String         @default("XOF")
  status            PurchaseStatus @default(PENDING)
  
  // Livraison
  downloadToken     String         @unique @default(uuid())
  downloadUrl       String?
  downloadExpiresAt DateTime?
  downloadCount     Int            @default(0)
  maxDownloads      Int            @default(3)
  
  // Upsell tracking
  upsellStatus      UpsellStatus   @default(NOT_CONTACTED)
  upsellAmount      Int?
  upsellNotes       String?
  
  // Monitoring
  adminNotified     Boolean        @default(false)
  
  createdAt         DateTime       @default(now())
  updatedAt         DateTime       @updatedAt

  @@index([email])
  @@index([phoneNumber])
  @@index([downloadToken])
}

// Composition du PDF
async function generateClientPDF(basePdfBuffer, clientData) {
  // 1. Chargement template base (pages fixes)
  const pdfDoc = await PDFDocument.load(basePdfBuffer)
  
  // 2. Injection données client
  // - Page 1 : Nom client, date
  // - Pages intermédiaires : Watermark email
  // - Page Devis : Calcul dynamique injection
  
  // 3. Génération QR Code unique
  const qrUrl = `https://wooplans.com/upsell/${clientData.token}`
  const qrImage = await generateQRCode(qrUrl)
  
  // 4. Page Upsell finale
  // - QR Code centré
  // - Texte "Plan Complet à partir de 150 000 FCFA"
  
  // 5. Watermark légal toutes pages
  // Footer : "Document d'étude - Non contractuel"
  // Footer : email client (traçabilité)
  
  return pdfDoc.save()
}

10. SÉCURITÉ
10.1 Protection des téléchargements
Tokens UUID : Non prédictibles (v4)
Expiration URL : 5 minutes (signée Bunny CDN)
Limite absolue : 3 downloads (compteur incrémenté atomiquement)
Watermark : Email client sur chaque page (dissuasion partage)
10.2 Données sensibles
Variables d'environnement pour toutes les clés API
Pas de logs console en production (ou sanitisation)
HTTPS obligatoire (cookies secure)
10.3 Validation entrées
Validation schéma Zod sur toutes les API routes
Rate limiting : 10 requêtes/minute par IP sur /api/purchase/*
Sanitisation emails (format, domaine)
