# Plan: Restructure Product Detail Page (Overlay Panel)

## Goal
Restructure the detail overlay panel in `index.html` (the product page that opens when clicking a plan card) to match the new structure provided by the user. The new design uses a single-column mobile-first layout with the existing warm cream color palette, dynamic Supabase data, and adds several new sections (countdown timer, offer section, testimonials, FAQ, related products, share buttons, bonus).

## Scope
- **File modified**: `index.html` only (the main catalogue/product page)
- **Target**: The `#detail-overlay` section and its associated CSS + JS
- **Theme**: Adapt the new dark theme to the existing warm cream palette (`--creme`, `--brun`, `--or`, etc.)
- **Data**: Keep dynamic from Supabase (existing data binding logic)

## Key Design Decisions
1. **Single-column layout**: The new structure is entirely single-column (max-width ~480px centered), which works well in the overlay panel context
2. **Cream theme adaptation**: All dark backgrounds become light cream, text colors become dark brown/grey
3. **Multiple CTAs**: The new design has a CTA button after each section (pricing, gallery, specs, offer) — all trigger `openCheckout()`
4. **Hero + Gallery split**: The first plan image is shown as a hero banner at the top. All images (including the hero image) are also shown in a masonry gallery grid in the second section, with lightbox, swipe, keyboard nav, and video support
5. **Masonry gallery**: CSS columns-based masonry layout for the image grid, with a lightbox overlay for full-size viewing
6. **Lightbox**: Full-screen overlay with nav arrows, keyboard support (←/→/Esc), swipe on mobile, video playback, image counter
7. **Countdown timer**: Client-side only, resets at midnight each day
8. **Testimonials**: Hardcoded text reviews (same as in the provided HTML), not from Supabase
9. **Related products**: Dynamically pulled from the existing `plans` array (same type, excluding current plan)
